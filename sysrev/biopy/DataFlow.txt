Basic Data Flow

1. Input Formatted Query.




2. Obtain the query and post it to Esearch, recieve UIDS.

   #ESearch

   #Functions

#Provides a list of UIDs matching a text query
#Posts the results of a search on the History server
#Downloads all UIDs from a dataset stored on the History server
#Combines or limits UID datasets stored on the History server
#Sorts sets of UIDs


#Required Parameters

          # ------db

#   Database to search. Value must be a valid Entrez database name (default = pubmed).


#-------------------------------------------------------------------------------

          #------term

#Entrez text query. All special characters must be URL encoded. Spaces may be replaced by '+' signs. Search fields and tags are database specific.


# for info on the other parameters refer to : http://www.ncbi.nlm.nih.gov/books/NBK25499/#chapter4.ESearch





3. (a) post to EGQuery (global query), recieve number of records matching the query.
   (b) post to Spell (spelling suggestions) recieve suggestions for the query submitted.
4.ECitMatch (batch citation searching in PubMed). Retrieves PubMed IDs (PMIDs) corresponding to a set of input citation strings.


5. Store UIDs somehow. (maybe Epost or to App database)




6. Obtain UIDs from storage. (maybe Efetch or from App Database)
7. Use UIDS to obtain summaries. (Esummary)




8. Links
